# iPAS AI 應用規劃師 備考 App — 設計文件

- 日期：2026-06-12
- 狀態：設計已確認，待撰寫實作計畫
- 作者：與使用者協作（brainstorming）

## 1. 目標與背景

協助使用者準備 iPAS「AI 應用規劃師」能力鑑定。將官方**簡章**、**學習指引**、**歷屆／樣題**整合進一款 App，提供「圖文重點摘要 + 線上測驗自動計分 + 錯題回顧」，並依級別／科目分類。

涵蓋級別與科目：
- **初級**（2 科）：人工智慧基礎概論、生成式 AI 應用與規劃
- **中級**（3 科）：人工智慧技術應用與規劃、大數據處理分析與應用、機器學習技術與應用

**首版範圍：初級 2 科**；架構設計為可無痛擴充至全 5 科。

## 2. 已確認的設計決策

| 主題 | 決策 |
|---|---|
| 平台型態 | 純前端**靜態 PWA**（可加到主畫面、離線使用），無後端、無帳號、無雲端費用 |
| 計分可行性 | 官方公告試題與樣題皆含「答案」欄位 → 做成真正的測驗＋自動計分 |
| 圖文摘要 | **預先生成**、固定內容、免費離線（不接 AI API） |
| 首版範圍 | 初級 2 科優先，架構支援擴充至中級 3 科 |
| 弱點分析 | **首版只做基本計分**；弱點章節（題目標籤）列為 Phase 2 |
| 個人紀錄 | 存於瀏覽器 localStorage，提供匯出／匯入 JSON 備份 |

## 3. 技術選型

- 框架：React + Vite + TypeScript
- 樣式：Tailwind CSS（響應式，手機／電腦自適應）
- 資料：題庫、科目設定、圖文摘要以 **JSON** 隨 App 打包（離線可讀）
- 個人紀錄：localStorage，可匯出／匯入 JSON
- PWA：manifest + service worker（加到主畫面、離線開啟）
- 部署：本機一鍵啟動使用；（選用）可免費部署至靜態主機（如 GitHub Pages），手機安裝為 PWA

### 資料夾結構（概念）

```
ipas-study-app/
├─ public/              # PWA icon、manifest
├─ src/
│  ├─ data/             # 題庫與摘要 JSON（依級別/科目分檔）
│  │   ├─ beginner/     #   初級兩科：題庫 + 摘要
│  │   └─ exam-info/    #   簡章/考試資訊
│  ├─ features/         # summary / quiz / scoring / wrong-book 等功能模組
│  ├─ components/       # 共用 UI 元件
│  ├─ store/            # localStorage 進度管理
│  └─ lib/              # scoreQuiz 等純函式
└─ tests/               # 計分邏輯、資料驗證測試
```

設計原則：**資料（題庫/摘要）與程式（功能）分離**。新增科目＝新增資料檔，不改架構。

## 4. 畫面結構與導覽

導覽三層：**級別 → 科目 → 功能**。手機底部 5 分頁、電腦左側欄。

| 畫面 | 用途 |
|---|---|
| 首頁 | 切換初級/中級、各科進度卡、最近成績、整體正確率 |
| 科目頁 | 該科圖文摘要章節清單 ＋「開始測驗」入口 |
| 摘要閱讀頁 | 單一章節重點（文字＋示意圖／表格／心智圖） |
| 測驗頁 | 練習模式（逐題即時對答案＋解析）／模擬考模式（依簡章題數時間、作答完一次計分） |
| 計分結果頁 | 分數、答對題數、是否通過、看解析、錯題入庫（弱點章節為 Phase 2） |
| 錯題本 | 自動收集做錯題目，可只挑錯題重練 |
| 考試資訊頁 | 從簡章整理：報名、費用、題型、及格標準、各科占比 |

## 5. 資料模型

### 題目

```json
{
  "id": "B1-Q001",
  "level": "beginner",
  "subject": "ai-basics",
  "source": "114-4-official",
  "stem": "在 AI 系統的決策流程中…",
  "options": [
    {"key": "A", "text": "…"},
    {"key": "B", "text": "…"},
    {"key": "C", "text": "…"},
    {"key": "D", "text": "…"}
  ],
  "answer": "B",
  "explanation": "（選填）解析…",
  "tags": []
}
```

`tags` 欄位首版可為空陣列；Phase 2 才填入章節標籤以支援弱點分析。

### 科目設定

```json
{
  "id": "ai-basics",
  "level": "beginner",
  "title": "人工智慧基礎概論",
  "examFormat": { "questionCount": 0, "minutes": 0, "passScore": 0 },
  "topics": []
}
```

`examFormat` 各數值（題數／時間／及格分數／配分）由 Phase 1 第一步從**簡章**擷取後填入；`topics`（各主題占比）對齊**評鑑內容範圍參考**。

### 圖文摘要

以章節為單位，內含區塊陣列：`text`（說明）、`keypoints`（重點條列）、`table`（比較表）、`image`（inline SVG 示意圖／流程圖／心智圖）。

### 個人紀錄（localStorage）

```json
{
  "version": 1,
  "subjects": {
    "ai-basics": {
      "summariesRead": ["ai-basics-ch1"],
      "quizHistory": [
        {"date": "…", "mode": "mock", "score": 82, "correct": 41, "total": 50}
      ],
      "wrongQuestions": ["B1-Q012"],
      "bookmarks": ["B1-Q005"]
    }
  }
}
```

## 6. 計分與測驗邏輯

- 純函式 `scoreQuiz(questions, answers, examFormat)` → `{ score, correct, total, pass, perQuestion }`
- 分數依簡章配分；`pass = score >= passScore`
- 練習模式：逐題即時比對，立即顯示對錯＋解析
- 模擬考模式：可打亂題序／選項順序（打亂選項時須同步維護正解對應）→ 全部作答後一次計分
- 錯題：計分後將答錯 id 寫入 `wrongQuestions`（去重）；錯題本可生成「只含錯題」的測驗
- 弱點分析（Phase 2）：每題依 `tags` 分組計算各主題正確率，低於門檻標記為弱點章節

## 7. 圖文摘要生成方式

- 由 Claude 依各科**學習指引**逐章整理，結構化為上述區塊
- 圖一律使用 inline SVG（清晰、檔案小、深色模式相容、離線可看）
- 對齊**評鑑內容範圍參考**，使涵蓋面貼近考試比重
- 內容忠於官方指引，定位為重點濃縮

## 8. 測試

- `scoreQuiz` 單元測試：答對數、及格門檻、邊界情況（全對／全錯／未作答）
- 資料驗證測試：每題答案需對應到存在的選項、id 不重複、必填欄位齊全 → 壞資料即讓測試失敗
- 主要使用流程煙霧測試

## 9. 建置階段

| 階段 | 內容 |
|---|---|
| Phase 1（首版可用） | 初級 2 科：簡章擷取 examFormat → 題庫數位化（公告試題＋樣題，含正解校對）→ 圖文摘要 → 練習＋模擬考 → 基本計分 → 錯題本 → 考試資訊頁 → PWA 離線安裝 |
| Phase 2 | 弱點章節分析（題目標籤）、紀錄匯出／匯入優化 |
| Phase 3 | 中級 3 科資料上架（純資料新增） |

主要工時集中於：①PDF 題目轉結構化 JSON（含正解校對）②各章圖文摘要整理。兩者皆為「填資料」，不影響架構。

## 10. 來源素材清單

- 簡章：114、115 年度 AI 應用規劃師能力鑑定簡章；評鑑內容範圍參考（11404）

- 中級：科目 1/2/3 學習指引、114 第二梯次公告試題（三科）、考試樣題（114.09 版 v2）
